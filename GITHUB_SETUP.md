# GitHub Setup for Quest Board

## Recommended Repo Name: `neuro-atlas`

---

## Step 1: Create Repository

1. Go to [github.com](https://github.com) → **New repository**
2. Name: `neuro-atlas` (or any name you like)
3. Visibility: **Public** (required for GitHub Pages free tier)
4. Check **Add README**
5. Click **Create repository**

---

## Step 2: Push Website Files

```bash
cd /path/to/your/candyland/folder

git init
git add .
git commit -m "feat: Quest Board v2 initial deploy"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/neuro-atlas.git
git push -u origin main
```

---

## Step 3: Enable GitHub Pages

1. Go to your repo → **Settings** → **Pages**
2. Source: **Deploy from a branch**
3. Branch: `main`, folder: `/ (root)`
4. Click **Save**
5. Your site will be live at: `https://YOUR_USERNAME.github.io/neuro-atlas/`

> Note: First deploy takes ~2 minutes. Subsequent pushes update the site within 30 seconds.

## Step 4: Daily Snapshot GitHub Action

Create `.github/workflows/daily_snapshot.yml`:

```yaml
name: Daily State Snapshot
on:
  schedule:
    - cron: '0 14 * * *'  # 6am PDT (UTC-8) every day
  workflow_dispatch:       # Also triggerable manually

jobs:
  snapshot:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Create snapshot placeholder
        run: |
          DATE=$(date +%Y-%m-%d)
          mkdir -p snapshots
          echo "{\"date\": \"$DATE\", \"note\": \"Daily checkpoint — update via Quest Board export\"}" > snapshots/snapshot_$DATE.json
      - name: Commit snapshot
        run: |
          git config user.name "Quest Board Bot"
          git config user.email "bot@questboard"
          git add snapshots/
          git diff --staged --quiet || git commit -m "snapshot: $DATE daily checkpoint"
          git push
```

## Step 5: Weekly Review GitHub Action

Create `.github/workflows/weekly_review.yml`:

```yaml
name: Weekly Sprint Planning Issue
on:
  schedule:
    - cron: '0 17 * * 1'  # 9am PDT Monday
  workflow_dispatch:

jobs:
  create-issue:
    runs-on: ubuntu-latest
    steps:
      - name: Create weekly issue
        uses: actions/github-script@v7
        with:
          script: |
            const date = new Date().toISOString().split('T')[0];
            await github.rest.issues.create({
              owner: context.repo.owner,
              repo: context.repo.repo,
              title: `Weekly Review — Week of ${date}`,
              body: `## Reflection — Past Week\n\n<!-- What did I complete? What stalled? What surprised me? -->\n\n## Plan — Next Week\n\n<!-- Top 3 priorities, which 90-min blocks am I committing to? -->\n\n## Triage Review\n\n<!-- Any Q1 items that became Q3? Any Q4 items you can drop? -->\n\n## Reprioritization\n\n<!-- Any changes to the order of milestones? New urgency? -->\n\n## XP Earned This Week\n\n<!-- Export from Quest Board and paste here -->\n\n---\n*Auto-generated by Quest Board Weekly Review action*`,
              labels: ['weekly-review']
            });
```

> **Tip:** Create the `weekly-review` label in your repo first: Issues → Labels → New label.

## Step 6: Google Sheets Integration

The tracker page supports live sync from a public Google Sheet.

1. Create a Google Sheet at [sheets.google.com](https://sheets.google.com)
2. Set up your BD2 Progress sheet with these columns:
   `Pair | Case | Dx | Area | Status | Prep | Imaging | Analysis | Sectioning | Staining | Mounting | Imaging Run | Registration | Cell Segmentation | Transcript Detection | Cell Typing | QC Passed | Automation Complete | Notes`
3. Share the sheet: **Share → Anyone with link → Viewer**
4. Copy the Sheet ID from the URL:
   `https://docs.google.com/spreadsheets/d/`**`SHEET_ID`**`/edit`
5. In Quest Board: open **📋 Tracker** → paste the full sheet URL → click **Sync**

The app fetches using the public JSON endpoint (no API key required):
```
https://docs.google.com/spreadsheets/d/SHEET_ID/gviz/tq?tqx=out:json&sheet=BD2+Progress
```

## Step 7: Priority Triage Workflow

The triage system classifies every task into a 2×2 matrix:

| | **Has due date** | **No due date** |
|---|---|---|
| **PhD Critical** | 🔴 Q1 — Do Now | 🟣 Q3 — Schedule |
| **Not PhD Critical** | 🟡 Q2 — Delegate | ⬜ Q4 — Drop |

### Daily habit (2 minutes):
1. Click the **Triage** button (bottom right) on the main board
2. Pick your 3 Q1 items for today — that's your maximum
3. The urgent strip at the top shows only those 3
4. Open Focus Mode → it auto-opens to your top Q1 task

### Lead time warnings:
For steps with compute/wet-lab lead time set (e.g., 24h for decoder run, 72h for IF staining):
- **Yellow banner**: start within 24 hours to meet deadline
- **Red banner**: last chance to start — must begin now
